Using templates

Wipeout viewModels alone do not have an appearance, for this we need templates. At it's most basic, a viewModel inherits from the class and the view class has a property.
Template id's reference the id of a html <script></script> tag with an xml template. It is not strictly html, as

  1. The content must be valid, strict XML.
  2. The content can contain wipeout viewModels which are not html tags.

In most cases, you will not want the the template of a viewModel to change. For this case you can set the template id in the constructor of the viewModel. var myViewModel = wo.view.extend(function myViewModel() { this._super('MyTemplateId'); // more constructor logic });
