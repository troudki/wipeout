View Models and Html Elements

Within wipeout templates, view models should be treated almost like html elements. They are defined in the same way and can sit side by side with elements

<div> <my-view-model></my-view-model> <input type="text" /> </div>

There are some differences in setting the properties of view models and html elements

<div> <my-view-model model="$model.value"></my-view-model> <input type="text" data-model="$model.value" wo-value="$model.value" /> </div>

In this example we have bound:

  1. The model of my-view-model to the value property of the model of the view model which the template was applied to.
  2. The data-model property of the input to the string "$model.value"
  3. The value of the input to the property of the model of the view model which the template was applied to. The difference between this case and the previous case is that is a .
