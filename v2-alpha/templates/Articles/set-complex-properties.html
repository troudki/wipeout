Set complex properties

Complex properties with multiple levels can also be set in xml. If there is no parser to go with the constructor property, then the constructor is used as an actual constructor with the new keyword

<wo.content> <!-- outer content --> <my-val constructor="wo.content" inner-value="'Hello wipeout'"> <!-- inner content --> <set-template> Binding to element value: <span data-bind="text: innerValue"></span> </set-template> </my-val> <set-template> Rendering viewModel: <div data-bind="render: myVal"></div> </set-template> </wo.content> <wo.view template-id="'set-complex-properties-script-1'" />

This is a fairly complex piece of functionality.

  1. We constructed an outer wo.contenentControl
  2. We constructed an inner wo.contenentControl
  3. We set the innerValue property of the inner content to "Hello Wipeout"
  4. We set the template property of the inner content.
  5. We set the myVal property of the outer content to the innercontent
  6. We set the template property of the outer content.
  7. Within the template of the outer content conrol, we use the binding to render its myVal property.
