Custom Parsers

You can add a custom parser by adding it to wo.parsers

// a parser which parses the value in the setter to // an xml doc wo.parsers.xmldoc = function (value, propertyName, renderContext) { // value: is the string value of the setter // propertyName: is the property name to be set // renderContext: is the current context, with $this, $parent etc... var xmlParser = new DOMParser(); return xmlParser.parseFromString(value, "text/xml"); }; // it is often a good idea to create an // alias so as not to clutter the template wo.parsers.xml = wo.parsers.xmldoc; Usage: <wo.view value--xml="<hello />"></wo.view>

Be careful not to use a parser when a will do. If you find you are parsing javascript code or that you also need to be able to set the value in the setter, you should use a filter.
