Add a default value

You can add a default value to any viewModel using:

  • value -or-
  • dynamicValue

Using value

value is used for primitive values. Primitive values are Strings, Numbers, Booleans, null and undefined. wo.viewModel("myNamespace.myViewModel") .value("myNumberProperty", 5) .value("myStringProperty", "five") .value("myBooleanProperty", true) .build();

In this case, when a myViewModel is created, it will have default properties like this:

{ myNumberProperty: 5, myStringProperty: "five", myBooleanProperty: true }

Using dynamicValue

dynamicValue is used for non primitive values. Non primitive values are Objects. dynamicValue is slightly modre complex than value, as it must create a new instance of the object and property each time a new viewModel is created.

wo.viewModel("myNamespace.myViewModel") .dynamicValue("myProperty", function () { return { innerValue: "hello" }; }) .build();

In this case, when a myViewModel is created, it will have default properties like this:

{ myProperty: { innerValue: "hello" } }
