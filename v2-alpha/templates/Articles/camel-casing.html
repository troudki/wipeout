camelCasing

Because HTML is not case sensitive, it is not advisable to use upper case letters in your element and attribute definitions. To get around this, wipeout supports camelCasing using the "-" symbol as the indicator of an upper case letter.
So for instance, the element &lt;app.my-view-model my-attribute="true"&gt;&lt;/app.my-view-model&gt; will be read as app.myViewModel with the attribute myAttribute="true".
You do not need to use this notation for values within an attribute, e.g. my-attribute="$parentContext.$this" is valid.
