View mode lifecycle

There are several events within the viewModel lifecycle which you may wish to execute code:

  1. initialize, will be run after the viewModel is initialized (i.e. it's values are bound from the template) but before it is rendered.
  2. rendered, will be run after each time the templateId changes and the viewModel is rendered.
  3. unRendered, will be run after each time the templateId changes and the viewModel is unRendered. unRender will be called before render.
  4. dispose, will be run when the viewModel is disposed of,
  5. initializeApplication is the same as initialize, but is only called if the viewModel has no parent viewModel.
wo.viewModel("myNamespace.myViewModel") .initialize(function () { // set up the viewModel // alter any values // define computeds }) .rendered(function () { // alter anything on templateItems }) .unRendered(function () { // dispose of anything which was created during "rendered" }) .dispose(function () { // dispose of anything created during "initialize" }) .initializeApplication(function () { // set up any application specific items or variables }) .build();
