The wipeout OO framework

Wipeout leverages a simple, lightweight object oriented model which enriches javascript code while not restricting it. This means you can use it to your advantage or you can hack it to bits (Disclaimer, don't hack it to bits) without breaking anything you wouldn't expect to break in javascript.

There are some things to note about the wipeout inheritance model.

  • Javascript is a prototypical language and so it is prototype inheritance which is used.
  • Wipeout does not embrace the the private/privileged js model. There are no private members, however properties which begin with an underscore should be treated as such
  • Inheritance is build to support instanceof checks.
  • Rember that setting the property on an object directly rather than it's prototype will override the entier inheritance chain.
