Define a viewModel

viewModel definitions create viewModel classes and determine how a viewModel will act once it is created by wipeout.

The most basic viewModel definition looks like this:

wo.viewModel("myNamespace.myViewModel").build();

What this definition will do is:

  1. Create a window.myNamespace object. If one exists already, do not overwrite it.
  2. Create a myViewModel function. myViewModel will be the viewModel class. In order to implement the most basic viewModel functionality (templates, lifecycle management etc...) the viewModel will inherit from by default.
  3. Append the myViewModel to window.myNamespace, (as window.myNamespace.myViewModel).
Always remember to build(...) viewModels after all definitions.
