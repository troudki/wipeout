Call a viewModel method

Using knockout event bindings always sets the context of your functions to the current scope. However, when using ancestors function, it is generally more useful to have ancestor obect as the scope. The $call API will do this.
You simply spell out what you want to do rather than using the regular js syntax. For example, in order to call the submit function on the model of the parent viewModel, passing in the value of true , you would use:
$call($parent).dot("model").dot("submit").args(true).
This method will ensure that the context of the submit function will be the model object. It will pass in the specified args as well as the expected args (such as the event args) passed in by knockout.

<wo.content> <set-template> <button data-bind="click: $call($parent).dot('model').dot('submit').args(true)">Submit</button> </set-template> </wo.content>
