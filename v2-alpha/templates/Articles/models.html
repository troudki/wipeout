Models

View models have a special property to deal with business logic and business data. This is the model property. The model is the driver of the application. It provides and recieves almost all data from both the server and the user. It is also the initial argumnent to which creates an application in the first place.

The model property is special in that:

  1. It has it's own alias within a template binding. Rather than use $this.model, you can simply use $model instead.
  2. Models cascade. If you do not explicitly set the model of a view model within the template or function of the wo.viewModel(...) tool then it will receive the model of it's parent.
<wo.content model--string="Hello wipeout!"> <set-template> <!-- notice how the model property is not set --> <wo.content> <set-template> $model: <span wo-content="$model"></span> <br /> $this.model: <span wo-content="$this.model"></span> <br /> $this.model === $model: <span wo-content="$this.model === $model"></span> <br /> $model === $parent.model: <span wo-content="$model === $parent.model"></span> <br /> </set-template> </wo.content> </set-template> </wo.content> <wo.view template-id="'models-example'"></wo.view>
