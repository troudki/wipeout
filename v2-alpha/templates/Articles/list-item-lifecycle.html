List item lifecycle

You can inherit from the list to create lists with tighter control on the view models they render.

With a custom list you can control and .

var mylist = wo.viewModel("mylist", wo.list) .build(); mylist.onItemRendered = function(item) { this._super(item); // do initialization logic here }; mylist.onItemRemoved = function(item) { this._super(item); // do cleanup logic here }; Usage: <my-list items="...">...</my-list>

Remember that onItemRendered and onItemRemoved are methods which already exist on the wo.list class. When you override a method of a parent class it is generally necessary to call the method so that the method of the parent class is still invoked.
