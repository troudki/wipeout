Bind to the property of an ancestor

There is a helper function which can find ancestors based on their properties, type, depth or any other custom attribute you need.

// create a custom class to distinguish type (inherit from content) wo.viewModel("demoApp.myType", wo.content) .build(); <demo-app.my-type model="{value: 'Hello wipeout!'}" value="123"> <set-template> <wo.content> <set-template> Found ancestor by type: <span wo-content="$context.find(demoApp.myType).model.value"></span> <br /> Found ancestor by hirearchy: <span wo-content="$context.find('parent').model.value"></span> <br /> Found ancestor by type: <span wo-content="$context.find({$instanceof: demoApp.myType}).model.value"></span> <br /> Found ancestor by property value: <span wo-content="$context.find({value: 123}).model.value"></span> <br /> Found ancestor model by property value: <span wo-content="$context.find({value: 'Hello wipeout!', $model: true}).value"></span> <br /> </set-template> </wo.content> </set-template> </demo-app.my-type> <wo.view template-id="'bind-to-ancestor-script-1'"></wo.view>

The value passed into $context.find can be

  1. A view model type
  2. A string indicating hirearchy (parent, grandparent, greatgrandparent etc...)
  3. An object of search parameters.
If search parameters are used, the available parameters are as follows:
Search type Alias Description
$instanceof $i Find an object which is an instanceof the given search term. Is the same as passing a viewModel type into $find
$type $t An exact type match. Is the same as currentViewModel.constructor === searchTerm.
$ancestory $a Find the parent, grandparent ect... Is the same as passing a string into the $context.find() method.
$number $n Describes when to stop when a viewModel is found. For example, if $n === 0, return the first match. If $n === 1, return the second match.
$model $m If set to true, signifies that we are looking for models, not viewModels.
anything else none Anything on the search object which is not prefixed with a $ will be treated as a property search. For instance using { myProperty: "hello"} will return the first viewModel where myProperty === hello .
