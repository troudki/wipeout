Html element/ViewModel name clashes

There may be some cases where wipeout view model and a html tag names can clash. For instance, if you were to set the value of window.div = function () { };. When this occurs, and the wipeout template engine sees a &lt;div> tag it will attempt to create an insance of window.div and render it.

To ensure this does not happen there is a full list of html tags at wo.definitelyNotAViewModel which are on a do not render list.

If you have a custom tag, e.g. &lt;my-custom-tag> which is definitely not a view model, you can add it to the list:

wo.definitelyNotAViewModel.myCustomTag = true;
