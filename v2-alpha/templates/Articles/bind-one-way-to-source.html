One way to source

By default, all bindings are one way from the parent (source) to the child (destination). This means that the object where the propery is set will recieve updates when the property changes, however, if the object where the property is set changes the value itself, the original owner of the property will not be notified. Sometimes you may wish to reverse this behavior, using the oneWayToSource binding. One way to source bindings use the --owts .

<wo.content> <set-template> One way to source bindings: <span wo-content="$model"></span> <br /> <wo.content input-value--owts="$model"> <set-template> <input type="text" wo-value="$this.inputValue" wo-on-event="keyup"/> </set-template> </wo.content> </set-template> </wo.content> <wo.view template-id="'bind-owts-script-1'"></wo.view>

Now if the child content control modifies it's model, the resulting change will reflect in the parent's model.value, but not view versa. To bind twa ways, you must use a binding.
